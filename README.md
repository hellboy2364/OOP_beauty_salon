# Информационная система «Салон красоты»
# 📌 О проекте
Разработка desktop-приложения для автоматизации работы салона красоты. Система предназначена для внутреннего использования персоналом (администраторами) и позволяет вести учёт клиентов, мастеров и записей на процедуры. Проект выполнен в рамках курсовой работы.

# Функциональные возможности
Управление клиентами
– Добавление нового клиента (имя, фамилия, телефон, email)
– Удаление клиента по ID
– Просмотр списка всех клиентов с количеством записей

Управление мастерами
– Добавление мастера с указанием специализации (парикмахер, маникюрист, педикюрист)
– Удаление мастера по ID
– Просмотр списка мастеров с контактными данными

Управление записями
– Создание записи на услугу с выбором клиента, мастера, даты, времени и типа процедуры
– Удаление записи по ID
– Автоматический расчёт стоимости услуги на основе типа мастера и выбранной процедуры
– Просмотр журнала записей

Хранение данных
– Все данные сохраняются в JSON-файлы (clients.json, masters.json, appointments.json)
– Загрузка данных при старте, сохранение при закрытии приложения
– Используется библиотека Newtonsoft.Json для сериализации/десериализации

# Технологии и инструменты
Язык программирования: C#

Платформа: .NET Framework / .NET Core (Windows Forms)

Библиотеки: Newtonsoft.Json

Тестирование: Unit-тесты (MSTest)

Архитектура: объектно-ориентированная с разделением на слои (модели, коллекции, менеджер данных, формы)

# Структура проекта
Salon.cs – главный класс-фасад, управляющий коллекциями и бизнес-логикой

Client.cs, Master.cs, Appointment.cs – модели данных

ClientCol.cs, MasterCol.cs, AppointCol.cs – коллекции для хранения объектов

dbManager.cs – работа с JSON-файлами (загрузка, сохранение)

Hairdresser.cs, Manicurist.cs, Pedicurist.cs – наследники Master с собственными услугами и ценами

MainForm.cs – главное окно с навигацией

ClientsForm.cs, MastersForm.cs, AppointForm.cs – формы для просмотра и управления

AddClientForm.cs, DelClientForm.cs и аналогичные – формы для добавления/удаления

UnitTest1.cs – модульные тесты для проверки основных операций

# Установка и запуск
1) Клонируйте репозиторий.

2) Откройте решение в Visual Studio (или другой совместимой IDE).

3) Установите пакет Newtonsoft.Json (если отсутствует):
Install-Package Newtonsoft.Json

4)Соберите проект.

5) Запустите приложение.

🧪 Тестирование
В проекте реализованы модульные тесты (MSTest), проверяющие:

добавление и удаление клиентов/мастеров

корректность сохранения и загрузки данных
